# AI 资讯获取与总结 - 最佳实践指南

## 核心原则

获取资讯并让AI总结，要遵循【分层处理】原则：

原始数据 → 筛选精简 → AI分析 → 结果输出

## 最佳实践方案

### 方案1: 专用API优先（最稳定）

适用场景: 美股、A股、财经数据等有专门API的领域

流程:
1. 调用腾讯财经/东方财富API获取结构化数据
2. 将精简后的数字给AI总结（几百tokens）
3. AI生成简要分析

优点:
- 数据准确、实时
- Token消耗极低（几百vs几千）
- 不受rate limit影响
- 速度快

### 方案2: 搜索+精简（通用方案）

适用场景: 没有专门API，必须搜索网页

流程:
1. 搜索获取网页内容
2. 关键步骤: 提取核心信息（不是全文给AI）
3. 将精简后的要点给AI总结
4. AI生成报告

Token优化技巧:

| 步骤 | 错误做法 | 正确做法 |
|------|----------|----------|
| 获取内容 | 全文复制 | 提取标题+关键段落 |
| 数据量 | 10,000字 | 500字要点 |
| 给AI的 | 原始网页 | 结构化数据 |
| Token | 5,000+ | 500-1000 |

### 方案3: 多源验证+摘要（高可靠）

适用场景: 重要资讯，需要确保准确性

流程:
1. 从3个不同来源获取同一话题资讯
2. 分别提取关键信息
3. 对比去重，整合要点
4. 给AI生成综合分析

## 稳定获取信息的关键机制

### 1. 分层处理架构

原始数据层（网页/API）
- 数据量大（10K+ tokens）
- 不稳定（可能失败）
↓ 提取、精简
精简数据层（要点）
- 数据量小（500-1000 tokens）
- 结构化（JSON/列表）
↓ AI分析
AI分析层
- 低token消耗
- 稳定输出
↓ 格式化
输出层（报告/推送）

### 2. 错误处理机制

- 主方案失败时自动切换备选方案
- 备选: 换模型（Kimi→DeepSeek）
- 备选: 返回原始要点（不经过AI）

### 3. Token控制清单

执行任务前检查：
- [ ] 输入数据是否已精简？
- [ ] 是否去除了无关内容？
- [ ] 是否使用了结构化格式？
- [ ] 预计token是否< 3K？
- [ ] 是否有备选方案？

## 具体实施建议

### 美股资讯获取（已实施）

采用方案1：专用API
- 腾讯财经API获取数字数据
- 返回结构化JSON
- Token消耗：~100
- 稳定性：极高

### 一般新闻资讯获取

建议采用方案2+3结合：
1. 搜索3个来源
2. 每个提取500字要点
3. 整合后给AI分析
4. 控制总token < 2K

## 监控指标

| 指标 | 安全范围 | 警戒 | 危险 |
|------|---------|------|------|
| Input tokens | < 3K | 3-5K | > 5K |
| 执行时间 | < 30s | 30-60s | > 60s |
| 失败率 | < 5% | 5-20% | > 20% |

## 总结

稳定获取资讯的黄金法则：

1. 能用API就不用搜索（专用API最稳定）
2. 能精简就不给全文（token控制是关键）
3. 能结构化就不给原始（JSON比HTML好）
4. 有备选就不怕失败（多模型/多方案）

一句话：
> 不要让AI读完整篇文章，让AI读你提取的要点。
