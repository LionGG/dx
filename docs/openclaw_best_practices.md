# OpenClaw 最佳实践核心要点

## 1. 代码执行原则

### 串行优于并行
- **顺序执行**：等一个任务完成再启动下一个，避免资源竞争
- **错峰启动**：不同任务间隔30秒-1分钟，减少数据库/CPU竞争
- **批量操作**：批量查询（100条/次）优于单条查询

### 停顿机制
- 每批次之间停顿 1-2 秒，给系统喘息时间
- 大批量操作时尤其重要（如5000+条记录）

## 2. 缓存策略

### 文件缓存
```python
# RSR缓存示例
CACHE_DIR = "/tmp/rsr_cache"
cache_file = os.path.join(CACHE_DIR, f"rsr_{date}.pkl")

# 读取缓存
if os.path.exists(cache_file):
    with open(cache_file, "rb") as f:
        return pickle.load(f)

# 保存缓存
with open(cache_file, "wb") as f:
    pickle.dump((rsr_20, rsr_252), f)
```

### 缓存优势
- 第1次计算耗时 2 分钟
- 后续直接读取，耗时 < 1 秒
- **节省 99% 时间**

## 3. 数据库优化

### 批量插入
```python
# 批量100条提交一次
for i in range(0, len(stocks), 100):
    batch = stocks[i:i+100]
    # 处理批次
    conn.commit()  # 每批提交一次
```

### 减少查询次数
- 避免在循环中反复查询数据库
- 先加载所需数据到内存，再处理

## 4. 监控与验证

### 进度汇报
- 每 50-100 条记录汇报一次进度
- 长时间任务（>2分钟）必须主动汇报
- 失败时立即报错，不要静默处理

### 数据验证
- 计算完成后立即统计验证
- 检查：总数、平均值、范围、分布
- 异常数据及时发现并修复

## 5. 脚本管理

### 版本控制
- 保留最终版本在工作目录
- 旧版本归档到 `archive/` 目录
- 添加 README.md 说明文件

### 命名规范
- `batch_optimized_v2.py` - 主要脚本
- `db_config.py` - 配置文件
- `calc_*.py` - 计算相关
- `fix_*.py` - 修复脚本

## 6. 错误处理

### 优雅降级
- 数据不足时返回 NULL，不是失败
- 单个股票错误不中断整个批次

### 日志记录
```python
try:
    # 操作
except Exception as e:
    print(f"错误 {stock_code}: {e}")
    # 继续处理下一个
```

## 7. 性能对比

| 方案 | 1800只股票 | 优化效果 |
|------|-----------|---------|
| 无缓存并行 | 60分钟 | 基准 |
| 有缓存串行 | 3-4分钟 | **节省 90%** |
| 批量100+停顿1秒 | 2-3分钟 | **最优** |

## 总结

> **串行 + 缓存 + 批量 + 停顿 = 高性能稳定执行**
