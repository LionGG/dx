# OpenClaw 四大技能学习总结

## 1. Self-Improving（自我改进）

**核心功能：**
- 让Agent记住执行中的错误并自我修正
- 用的越多逻辑越顺
- 具备"Review"机制，每次任务后自我反思

**工作原理：**
- 分析用户交互习惯，自动微调AI响应逻辑
- 记录错误和优化方案到经验库
- 避免重复犯同样的错误

**关键文件：**
- `AGENTS.md` - 记录操作规则和经验教训
- `ERROR_PREVENTION.md` - 记录已纠正的错误
- 每日笔记 - 记录原始执行过程

**使用场景：**
- 定时任务执行后自动检查错误
- 根据历史经验优化执行流程
- 建立个人化的执行模式

---

## 2. Proactive-Agent（主动规划）

**核心功能：**
- 主动拆解和多步任务权限
- 不等待指令，主动创造价值
- 自主规划任务流程

**核心哲学：**
> "不要问'我该做什么？'，而要问'什么能真正让我的用户惊喜？'"

**主动行为：**
- 预测需求：在需求表达前就预判
- 反向提示：提出用户没想到的想法
- 主动检查：监控重要事项，需要时主动联系
- 像主人一样思考，不像员工一样等待

**关键机制：**
- **Heartbeat系统** - 定期自我检查
- **Reverse Prompting** - 主动提出建议
- **Growth Loops** - 持续成长循环

**Heartbeat检查清单：**
- 安全检查（注入尝试扫描）
- 自我修复（日志审查、问题修复）
- 主动行为（有什么可以帮用户的？）
- 记忆维护（更新MEMORY.md）

---

## 3. Memory-Setup（长期记忆）

**核心功能：**
- 跨对话记住核心偏好和关键信息
- 构建长期记忆架构
- 避免每次对话都从零开始

**记忆架构：**
```
workspace/
├── ONBOARDING.md      # 首次运行设置
├── AGENTS.md          # 操作规则和经验
├── SOUL.md            # 身份、原则、边界
├── USER.md            # 用户上下文、目标、偏好
├── MEMORY.md          # 精选长期记忆
├── SESSION-STATE.md   # 活跃工作记忆
├── HEARTBEAT.md       # 定期自我改进清单
└── memory/
    ├── YYYY-MM-DD.md  # 每日原始记录
    └── working-buffer.md  # 危险区域日志
```

**关键原则：**
- **Text > Brain** - 记忆有限，要写到文件
- 用户说"记住这个" → 写入每日笔记
- 学到教训 → 更新AGENTS.md
- 犯错 → 记录以防再犯

**记忆维护：**
- 定期回顾每日笔记
- 更新MEMORY.md（精选学习）
- 删除过时信息

## 4. Automation-Workflows（工作流编排）

**核心功能：**
- 把多个技能串起来干更复杂的工作流
- 定时触发任务（cron jobs）
- 自动化执行复杂流程

**工作流模式：**
- **定时触发** - cron定时执行
- **事件触发** - 特定条件触发
- **手动触发** - 用户指令触发

**典型工作流示例：**

### 股市数据工作流（我们的场景）
```
16:05 个股数据抓取
  ↓
16:10 MA50占比计算
  ↓
16:20 短线情绪数据抓取
  ↓
16:25 短线情绪研判（AI分析）
  ↓
16:30 短线情绪部署通知
  ↓
发送飞书通知
```

### 其他常见工作流
- **每日资讯** - 搜索→整理→发送
- **文档清理** - 扫描→归档→删除
- **数据备份** - 检查→备份→验证

**工作流设计原则：**
1. 范围要窄 - 一个工作流只做一件事
2. 先只读后写入 - 先验证逻辑再执行
3. 破坏性操作要确认 - 删除、修改需要显式确认
4. 配置和代码分离 - 路径等放配置，不放代码

## 关键学习点

1. **不要等待指令** - 主动思考什么对用户有帮助
2. **写下来** - 记忆有限，文件才是持久的
3. **持续改进** - 每次执行都比上次更好
4. **系统化** - 把工作流程化、自动化
5. **验证再报告** - 说"完成"前要先测试
