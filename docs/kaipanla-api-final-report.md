# 开盘啦(Kaipanla)数据接口研究报告 - 最终版

## 一、接口发现

### 1.1 真实API地址
| 类型 | URL |
|------|-----|
| 行情数据 | `https://pchq.kaipanla.com/w1/api/index.php` |
| 龙虎榜数据 | `https://pclhb.kaipanla.com/w1/api/index.php` |

### 1.2 请求方式
- **Method**: POST
- **Content-Type**: application/x-www-form-urlencoded
- **Headers**: User-Agent, Referer

### 1.3 核心接口

#### 市场情绪数据
```python
param = {
    'c': 'PCArrangeData',
    'a': 'GetIndexPlate',
    'st': 3,
    'StockID': 'SH000001',  # 上证指数
    'UserID': 399083,       # 用户ID
    'Token': 'xxx'          # 需要有效Token
}
```

**返回数据字段：**
| 字段 | 含义 | 用途 |
|------|------|------|
| ZRZT | 自然涨停 | 情绪强度 |
| ZTZB | 涨停炸板 | 情绪转折 |
| SZJS | 上涨家数 | 市场广度 |
| XDJS | 下跌家数 | 市场广度 |
| ZT | 涨停家数 | 情绪热度 |
| DT | 跌停家数 | 恐慌程度 |
| ZBL | 炸板率 | 接力意愿 |
| CGL | 昨日打板成功率 | 赚钱效应 |
| YLL | 昨日打板盈利率 | 赚钱效应 |

## 三、使用限制

### 3.1 必须解决
1. **Token认证**: 所有接口都需要有效的UserID和Token
2. **Token有效期**: Token会过期，需要定期更新
3. **获取方式**: 必须通过抓包APP或逆向工程获取

### 3.2 风险
- APP更新可能改变接口
- Token可能被封禁
- 非官方API，无保障

## 五、建议方案

### 方案A: 完整抓包（推荐）
**步骤：**
1. 安装Fiddler/Charles
2. 配置手机代理
3. 安装HTTPS证书
4. 打开开盘啦APP抓包
5. 获取UserID和Token
6. 编写自动化脚本

**优点：** 数据最全、实时
**缺点：** 需要抓包环境、Token可能过期

### 方案B: 使用已有Token（临时）
**步骤：**
1. 搜索GitHub/论坛找可用Token
2. 测试Token有效性
3. 定期更换Token

**优点：** 快速可用
**缺点：** Token不稳定、可能被封

### 方案C: 替代方案（保底）
**使用新浪财经API + 数据库已有数据**
- 新浪财经：实时指数、成交额
- 数据库：情绪数据、历史数据

**优点：** 稳定、免费
**缺点：** 数据不如开盘啦丰富
