## 🎯 今日重大变更

### 1. 持仓数据存储方式升级

**变更前**:
- 持仓数据分散在多个markdown文件
- 查找困难，难以对比分析
- 位置: `memory/trade/holdings/`、`memory/2026-XX-XX.md`

**变更后**:
- 所有持仓数据结构化存入SQLite数据库
- 支持SQL查询、跨日期对比
- 位置: `stock-plan/data/portfolio.db`

**数据范围**:
- 2月24日 - 3月10日（11个交易日）
- 3个账户（A-杠杆/B-均衡/C-核心）
- 33条持仓快照 + 140+条个股记录

---

### 2. 新增数据表结构

| 表名 | 用途 | 记录数 |
|------|------|--------|
| `portfolio_snapshots` | 持仓快照（总资产、盈亏、杠杆） | 33条 |
| `stock_holdings` | 个股持仓明细（代码、仓位、盈亏） | 140+条 |
| `trade_records` | 交易记录（买入/卖出/加仓/减仓） | 14条 |
| `trade_plans` | 交易计划 | 4条 |
| `trade_reviews` | 点评/分析报告 | 5条 |
| `psychology_logs` | 心情/心理状态记录 | 9条 |

---

### 3. 新建Portfolio Parser Skill

**位置**: `skills/portfolio-parser/`

**功能**:
1. 自动解析持仓截图
2. 提取结构化数据（总资产、个股、仓位、盈亏）
3. 存入数据库
4. 对比昨日生成变化报告

**使用方法**:
```python
from portfolio_parser import save_to_database, generate_report

# AI解析图片后保存
save_to_database(parsed_data)

# 生成对比报告
report = generate_report('A', '2026-03-10')
```

---

### 4. 查询工具

**位置**: `stock-plan/scripts/query_portfolio.py`

**功能**:
- 查询某日持仓: `--account A --date 2026-03-10`
- 对比两个日期: `--account A --compare 2026-03-03,2026-03-10`
- 查询某股票历史: `--stock 300059 --start 2026-02-24 --end 2026-03-10`

---

### 5. 后续工作流程（新规范）

**每日收盘后**:
1. 宝总发送持仓截图（飞书/Kimi）
2. AI解析图片内容
3. 自动提取数据 → 存入数据库
4. 对比昨日生成变化报告
5. 记录交易计划、心情、随想

**数据流向**:
```
持仓截图 → AI解析 → 存入数据库 → 对比生成报告
    ↓
交易计划/心情/随想 → 实时存入对应表
```
