## 📋 已纠正的错误清单

### 1. Gateway 重启问题
**错误**: 频繁重启 Gateway 导致不稳定
**纠正**: 
- 避免不必要的重启
- 配置变更保存即可，不立即重启
- 统一在周日 03:00 自动重启
**检查点**: 重启前确认必要性

### 2. 配置文件损坏
**错误**: 编辑配置时格式错误导致 Gateway 无法启动
**纠正**:
- 修改前总是备份
- 修改后验证 JSON 格式
- 保留多个备份版本
**检查点**: python3 -c "import json; json.load(open('config'))"

### 3. 日记任务缺失
**错误**: 日记任务执行了但没有生成文件
**纠正**:
- 改为 exec 类型直接执行脚本
- 不再依赖 systemEvent
**检查点**: 检查日记文件是否生成

### 4. 敏感信息硬编码
**错误**: 密码、Token 明文写在脚本中
**纠正**:
- 统一使用 Secrets 管理
- 所有脚本改用 secrets_manager 模块
**检查点**: grep 检查脚本中是否有硬编码密码

### 5. 任务隔离不足
**错误**: 所有任务在 main 会话执行，失败互相影响
**纠正**:
- 数据/分析类任务改为 isolated 模式
- 只有交互类任务保持 main
**检查点**: 检查 sessionTarget 配置

### 6. 定时任务配置不完整
**日期**: 2026-03-05
**错误**: 只改了一半配置（如只改agentId没改sessionTarget），导致任务跳过或失败
**影响**: 薄荷日记、量化资讯等任务未执行或内容错误
**纠正**:
- 批量修复所有任务的配置一致性
- agentId=main 必须配 sessionTarget=main
- isolated 必须配 agentTurn，main 必须配 systemEvent
**预防措施**:
- 改完配置后全面核查所有相关字段
- 使用验证脚本检查配置一致性
- 改完立即测试，不等到定时执行
**检查点**: python3 验证脚本检查所有任务配置

### 7. 脚本内容未验证

**影响**: 量化资讯还是推送了旧内容（含市场数据）
**纠正**:
- 改完脚本立即手动运行验证
- 检查输出内容是否符合要求
- 确认发送目标正确（个人/群）
**预防措施**:
- 脚本修改后必须手动测试
- 测试输出内容、格式、发送目标
