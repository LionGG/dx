## 📋 已纠正的错误清单

### 1. Gateway 重启问题
**错误**: 频繁重启 Gateway 导致不稳定
**纠正**: 
- 避免不必要的重启
- 配置变更保存即可，不立即重启
- 统一在周日 03:00 自动重启
**检查点**: 重启前确认必要性

### 2. 配置文件损坏
**错误**: 编辑配置时格式错误导致 Gateway 无法启动
**纠正**:
- 修改前总是备份
- 修改后验证 JSON 格式
- 保留多个备份版本
**检查点**: python3 -c "import json; json.load(open('config'))"

### 3. 日记任务缺失
**错误**: 日记任务执行了但没有生成文件
**纠正**:
- 改为 exec 类型直接执行脚本
- 不再依赖 systemEvent
**检查点**: 检查日记文件是否生成

### 4. 敏感信息硬编码
**错误**: 密码、Token 明文写在脚本中
**纠正**:
- 统一使用 Secrets 管理
- 所有脚本改用 secrets_manager 模块
**检查点**: grep 检查脚本中是否有硬编码密码

### 5. 任务隔离不足
**错误**: 所有任务在 main 会话执行，失败互相影响
**纠正**:
- 数据/分析类任务改为 isolated 模式
- 只有交互类任务保持 main
**检查点**: 检查 sessionTarget 配置

## 🔍 防重复犯错检查清单

### 执行任何操作前，检查：
- [ ] 是否做过类似操作？查本文件
- [ ] 上次怎么纠正的？按纠正方法执行
- [ ] 是否需要备份？敏感操作必须备份
- [ ] 是否有验证步骤？执行后验证结果

### 常见操作的标准流程：

#### 修改配置文件
1. 备份原文件
2. 修改并验证格式
3. 保存但不立即重启
4. 等待计划重启时间

#### 重启 Gateway
1. 确认必要性
2. 检查当前任务状态
3. 选择合适时间（避免交易时段）
4. 重启后验证状态

#### 修改定时任务
1. 检查现有任务列表
2. 避免重复创建
3. 测试脚本先手动执行
4. 配置后检查语法

### 6. 定时任务配置不完整
**日期**: 2026-03-05
**错误**: 只改了一半配置（如只改agentId没改sessionTarget），导致任务跳过或失败
**影响**: 薄荷日记、量化资讯等任务未执行或内容错误
**纠正**:
- 批量修复所有任务的配置一致性
- agentId=main 必须配 sessionTarget=main
- isolated 必须配 agentTurn，main 必须配 systemEvent
**预防措施**:
- 改完配置后全面核查所有相关字段
- 使用验证脚本检查配置一致性
- 改完立即测试，不等到定时执行
**检查点**: python3 验证脚本检查所有任务配置

### 7. 脚本内容未验证
**日期**: 2026-03-05
**错误**: 改完脚本没测试，直接等定时任务执行
**影响**: 量化资讯还是推送了旧内容（含市场数据）
**纠正**:
- 改完脚本立即手动运行验证
- 检查输出内容是否符合要求
- 确认发送目标正确（个人/群）
**预防措施**:
- 脚本修改后必须手动测试
- 测试输出内容、格式、发送目标
- 不测试不部署
**检查点**: 手动运行脚本，检查输出

## ✅ 改完配置/脚本后的标准检查流程

### 必须执行的检查：
1. **格式检查**: python3 -c "import json; json.load(open('config'))"
2. **字段核对**: agentId、sessionTarget、payload.kind 必须一致
3. **内容验证**: 脚本输出符合要求（无市场数据、无建议等）
4. **目标确认**: 推送给个人还是群，配置正确
5. **功能测试**: 手动运行一遍，确认无误

### 检查通过标准：
- [ ] JSON格式正确
- [ ] 所有字段配置一致
- [ ] 脚本内容符合要求
- [ ] 手动测试通过
- [ ] 发送目标正确

**不检查完，不说"改好了"**
