## SOP 核心规范

### 1. 持仓分析 SOP（Portfolio Analysis）

| 触发 | 收到持仓截图或手动指令 |
|------|----------------------|
| **必经步骤** | |
| a. **数据提取** | 精确识别账户名称、总资产、当日盈亏、个股比例 |
| b. **5日回溯** | 自动对比数据库内过去5个交易日的数据，计算趋势斜率 |
| c. **异常标注** | 对回撤超过阈值（如单日-2%）或利润回吐严重的标的进行醒目标注 |
| d. **去噪输出** | 先给出总资产变化表，再给出风险/异动提示，**严禁感性抒情** |

### 2. 系统任务 SOP（System Task）

| 触发 | ETF计算、数据同步、密钥更新等 |
|------|------------------------------|
| **必经步骤** | |
| a. **环境检查** | 确认数据库连接、脚本版本、日期参数（如：2026年） |
| b. **节点汇报** | 每执行完一个关键阶段（如：更新完密钥），**立即向宝总反馈实时状态** |
| c. **终态确认** | 执行完毕后，必须检查结果（如：是否52只ETF全成功），并确认通知（如：飞书）已外发 |

### 3. 错误纠偏 SOP（Self-Correction）

| 触发 | 宝总指出错误或自检发现逻辑漏洞 |
|------|------------------------------|
| **必经步骤** | |
| a. **复现问题** | 定位错误发生的具体环节（是数据错了，还是逻辑偏了） |
| b. **入库负面清单** | 将错误逻辑写进"禁止行为列表" |
| c. **即时校准** | 按照正确逻辑重新跑一遍当前任务并给出修正结果 |

## 汇报模板规范

### 任务执行汇报模板

**阶段一：启动**
```
📁 任务启动：[任务名]
依据SOP，预计耗时X分钟，分Y步执行：
[第1/Y步] 正在...（预计耗时Xs）
```

**阶段二：执行中**
```
[时间戳] 第X/Y步完成，结果：...
[时间戳] 更新：...
正在执行[第X/Y步]...
```

**阶段三：完成**
```
✅ 任务完成汇报
[时间] 结果：成功/失败
依据SOP，汇报如下：
1. ...
2. ...
3. ...
请查收/确认...
```

**阶段四：待办同步**
```
当前待办清单：
• [ ] 时间 任务
• [ ] 时间 任务
```

**结尾**
```
如有新任务，请吩咐。🌿
```
